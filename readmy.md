# Твой путь к «кодовому убийце» (2026–2027)

**Текущий уровень:** Codewars 6 kyu (в основном JS, знаешь TS/JS/PHP/C поверхностно). Есть базовое мышление, слабые алгоритмы и структуры данных, мало глубины в системах и практики.

**Правила на весь план:**
- Без подсказок, StackOverflow и готового кода — только официальная документация, книги и свои решения.
- Застрял — разбирай ошибку вручную (gdb/valgrind для C, console.log для JS и т.п.), потом иди дальше.
- Чтение чужого кода каждый день: **прочитал → закрыл → переписал сам → сравнил**.
- Рефакторинг: каждые 7–10 дней переписывай старый код короче и проще.
- Языки: чередуй JS/TS/PHP с C (для глубины памяти и систем).
- Веди лог: что сделал, что сломал, что понял.
- Если комфортно — усложняй (например, реализуй без stdlib).

**Баланс:** теория (книги, доки) даёт «почему и как», практика (реализация, задачи, проекты) — основная часть работы. Детальные планы по неделям лежат в папках `месяц / N неделя /`.

**Дни и месяцы** в плане — это **последовательность этапов**, а не календарь. Ты в плане укладываешься, когда делаешь шаги по порядку; темп свой. «Дни 1–7» значит «первый блок», «месяц 1–2» — «первый большой этап». Можешь растянуть блок на две недели или пройти быстрее — важно не пропускать шаги и не переходить к следующему этапу, пока не закрыл предыдущий.

## Месяц 1–2: Базовый фундамент

**Цель:** переход от 6 kyu к уверенному medium, базовые структуры данных и алгоритмы в голове и в коде.

### Дни 1–7
- [ ] **Теория:** «Introduction to Algorithms» (CLRS) — главы 1–3, 10: массивы, списки, стеки, очереди (ключевые разделы).
- [ ] **Практика:** Реализовать массив, список, стек, очередь вручную в JS/PHP и при желании в C (без библиотек).
- [ ] **Задачи:** Codewars 6–5 kyu: arrays, strings, fundamentals (объём по плану недели, см. `месяц/1 неделя/`).
- [ ] **Чужой код:** Каждый день — один модуль или 40–60 строк: прочитать → закрыть → переписать сам → сравнить.
- **Итог:** Базовые DS реализованы, 7–10+ kata, привычка к циклу «теория → своя реализация → задачи → чужой код».

### Дни 8–14
- [ ] **Теория:** CLRS — главы 6–7: хэш-таблицы, деревья.
- [ ] **Практика:** Переписать связный список и сделать простую hash map в C (или в JS/PHP).
- [ ] **Задачи:** Codewars 5–4 kyu (без подсказок).
- [ ] **Чужой код:** SQLite — первые 500 строк `src/main.c`: прочитать → переписать ключевую логику сам → сравнить.
- **Итог:** Понимание хэш-таблиц и деревьев, ещё 10 kata.

### Дни 15–30
- [ ] **Теория:** CLRS — глава 22: графы.
- [ ] **Практика:** Реализовать BFS и DFS вручную (на выбранном языке).
- [ ] **Задачи:** Codewars 4 kyu (5–10 kata).
- [ ] **Чужой код:** Redis — `src/server.c`, порядка 300 строк: прочитать → переписать модуль сам → сравнить.
- **Итог:** Графы и обходы в коде, 5–10 kata 4 kyu.

### Дни 31–60
- [ ] **Теория:** Competitive Programmer's Handbook — главы 1–5.
- [ ] **Практика:** Codewars/Codeforces div 2 A/B; свой stack + queue в C, протестировать на 100k элементов.
- [ ] **Чужой код:** Linux kernel — `fs/ext4/inode.c`, порядка 400 строк: прочитать → переписать фрагмент сам → сравнить.
- **Итог:** 50–70 kata за два месяца, базовые DS и алгоритмы уверенно в руках.

## Месяц 3–4: Алгоритмы и паттерны

**Цель:** динамика и жадные алгоритмы, первые паттерны проектирования, рост в задачах.

### Дни 1–15
- [ ] **Теория:** CLRS — главы 15–16: динамическое программирование, жадные алгоритмы.
- [ ] **Практика:** Реализовать knapsack, longest common subsequence в JS/C (или PHP).
- [ ] **Задачи:** Codeforces div 2 B/C.
- [ ] **Чужой код:** Redis — `src/ae.c` (event loop): прочитать → переписать идею сам → сравнить.
- **Итог:** Понимание ДП и жадных, рост рейтинга на Codeforces.

### Дни 16–30
- [ ] **Теория:** «Design Patterns: Elements of Reusable Object-Oriented Software» — главы 1–3 (factory, singleton, observer).
- [ ] **Практика:** Применить паттерны в старом коде из месяца 1, затем упростить или убрать лишнее.
- [ ] **Задачи:** LeetCode easy/medium (зарегистрироваться и решать вручную).
- **Итог:** 20–30 задач LeetCode, паттерны не только знаешь, но и применяешь.

### Дни 31–60
- [ ] **Теория:** «Head First Design Patterns» — главы 4–6: strategy, adapter.
- [ ] **Практика:** Мини-проект — свой event emitter в JS (как observer), использовать в задаче.
- [ ] **Задачи:** Codeforces div 2 C/D.
- [ ] **Чужой код:** SQLite — `src/vdbe.c`, порядка 500 строк: прочитать → переписать фрагмент → сравнить.
- **Итог:** Рейтинг Codeforces ~1400–1600, паттерны в применении.

## Месяц 5–6: Системы и concurrency

**Цель:** память, процессор, виртуальная память, многопоточность — код, который не ломается.

### Дни 1–15
- [ ] **Теория:** «Computer Systems: A Programmer's Perspective» (CS:APP) — главы 1–3: память, процессор.
- [ ] **Практика:** Задачи на bit manipulation в C (сдвиги, маски); LeetCode medium (bit, dp).
- [ ] **Чужой код:** Linux kernel — `mm/page_alloc.c`, порядка 400 строк.
- **Итог:** Понимание представления данных и памяти.

### Дни 16–30
- [ ] **Теория:** CS:APP — главы 9–10: виртуальная память.
- [ ] **Практика:** Написать простой allocator в C; Codeforces div 1 A.
- [ ] **Чужой код:** Nginx — `src/http/ngx_http_request.c`: прочитать → переписать модуль → сравнить.
- **Итог:** Понимание выделения памяти, рост в задачах.

### Дни 31–60
- [ ] **Теория:** «C++ Concurrency in Action» — главы 1–4 (или Rust docs по concurrency, если перейдёшь).
- [ ] **Практика:** Thread pool в C (pthread), lock-free очередь; мини-проект — многопоточный HTTP-сервер на C.
- [ ] **Окружение:** Docker — запускать свой HTTP-сервер и при необходимости БД в контейнерах (Dockerfile, docker-compose). Официальная дока Docker: образ, контейнер, сеть.
- [ ] **Практика (real-time):** WebSockets — простой сервис с реальным временем (чат, уведомления или обновления данных) на JS/Node или поверх своего HTTP-сервера. Понять handshake, кадры, поддержание соединения.
- [ ] **Чужой код:** PostgreSQL — `src/backend/access/nbtree/`, порядка 500 строк.
- **Итог:** Понимание памяти и concurrency, сервер в контейнере, первый опыт WebSockets.

**Цель:** чистая архитектура, DDD, разбор больших кодовых баз.

### Дни 1–30
- [ ] **Теория:** «Clean Architecture» — главы 1–10.
- [ ] **Практика:** Переписать старый проект (из месяца 2) с чистой архитектурой/DDD, затем упростить.
- [ ] **Задачи:** LeetCode hard (15–20).
- [ ] **Чужой код:** React — `packages/react/src/ReactHooks.js`: прочитать → переписать логику → сравнить.
- **Итог:** Архитектура в голове, код масштабируемый.

### Дни 31–90
- [ ] **Теория:** «Domain-Driven Design» — первые 150 страниц.
- [ ] **Практика:** Большой проект — мини-база (key-value + индексы) в C/JS.
- [ ] **Окружение:** Собирать и запускать проект (и зависимости) через Docker; при необходимости docker-compose для нескольких сервисов.
- [ ] **Задачи:** Codeforces div 1 B.
- [ ] **Чужой код:** Linux kernel — `net/ipv4/`, порядка 600 строк.
- [ ] **Open-source:** Найти баг в маленьком репо на GitHub, починить вручную (PR пока не обязателен).
- **Итог:** Проекты как у сильного инженера, первый вклад в чужой код, привычный запуск через Docker.

## Месяц 10–12: Глубокое погружение

**Цель:** сложные проекты, рефакторинг legacy, уровень staff.

### Дни 1–30
- [ ] **Теория:** «A Philosophy of Software Design» — вся книга.
- [ ] **Практика:** Интерпретатор маленького языка (типа Brainfuck+) на C; рефакторинг — сократить на 50% код из предыдущих проектов.
- [ ] **Задачи:** LeetCode/Codeforces hard, div 1.
- **Итог:** Умение упрощать и переосмыслять код.

### Дни 31–90
- [ ] **Практика:** Большой проект — распределённая очередь (мини-Kafka на C).
- [ ] **Очереди в проде:** RabbitMQ или Kafka — поставить (лучше в Docker), прочитать официальную доку, написать producer и consumer (отправка и приём сообщений), один сценарий (например задача в очередь → обработчик). Сравнить модель и гарантии со своей мини-очередью.
- [ ] **Чужой код:** PostgreSQL — `src/backend/optimizer/`: прочитать → переписать фрагменты → сравнить.
- [ ] **Open-source:** Внести код в средний репо (Redis-like).
- **Итог:** Рейтинг Codeforces 1800+, проекты уровня staff-engineer, опыт message broker (RabbitMQ/Kafka).

## Месяц 13–18: Legacy и менторство

**Цель:** рефакторинг legacy, компиляторы/GC, вклады в большие репо, объяснение сложного.

- [ ] **Практика:** Каждый день — legacy-код (свой старый или с GitHub), рефакторинг в 2–3 раза короче.
- [ ] **Проекты:** Свой компилятор или свой GC.
- [ ] **Open-source:** PR в большие репо (Linux, PostgreSQL).
- [ ] **Менторство:** Блог или видео — объяснять сложную тему словами, без кода.
- [ ] **Задачи:** Codeforces div 1 C/D.
- **Итог:** Решаешь «невозможное», код простой и мощный, другие учатся у тебя.

## Общие правила на всё время

- Если застрял на шаге — пропусти, вернись через неделю.
- Раз в 6 месяцев оценивай себя по Codeforces/LeetCode: если рост остановился — увеличивай долю практики (больше задач, больше своего кода).
- Детальные планы по дням и задачам — в папках `месяц / N неделя /` (например, первая неделя: `месяц/1 неделя/week-1-plan.md`).
